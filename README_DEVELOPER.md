# ClawSetup Developer Guide

Welcome to the ClawSetup repository! This document provides instructions for developers who want to modify, extend, or build the ClawSetup installer from source.

## How to Run from Source

1.  **Clone the Repository:**
    ```bash
    git clone https://github.com/openhands/openclaw.git
    cd openclaw/ClawSetup
    ```

2.  **Install Dependencies:**
    Ensure you have Python 3.10+ installed.
    ```bash
    pip install -r requirements.txt
    ```

3.  **Run the Application:**
    ```bash
    python main.py
    ```

## How to Add New Agents

1.  **Create the JSON Template:**
    Navigate to `templates/agents/` and create a new JSON file (e.g., `new_agent.json`). Follow the schema defined in the existing agent files.

2.  **Update the GUI Selector:**
    Open `gui/screen_agent_selector.py`. Add your new agent to the `self.agents` dictionary. The key should match the filename (without `.json`), and the value should be a descriptive string.

    ```python
    self.agents = {
        # ... existing agents ...
        "new_agent": "New Agent - Description of what it does."
    }
    ```

## How to Add New Screens

1.  **Create the Screen Class:**
    Create a new Python file in the `gui/` directory (e.g., `screen_custom.py`).
    Define a class that inherits from `ctk.CTkFrame`.

    ```python
    import customtkinter as ctk

    class ScreenCustom(ctk.CTkFrame):
        def __init__(self, master, on_next, on_back):
            super().__init__(master, fg_color="transparent")
            self.on_next = on_next
            self.on_back = on_back
            # ... add your widgets ...
    ```

2.  **Integrate into `main.py`:**
    Open `main.py` and import your new screen. Add it to the `self.screens` list in the `ClawSetupApp` class. Ensure the order is correct.

## How to Build for Distribution

We use PyInstaller to package the application into standalone executables.

*   **Windows:**
    Run `build_windows.bat` from the command prompt. This will create a single `.exe` file in the `dist/` folder.

*   **macOS:**
    Run `bash build_mac.sh` from the terminal. This will create an `.app` bundle in the `dist/` folder. Note the comments in the script regarding code signing.

## Environment Variable Reference

The `.env` file generated by ClawSetup configures the OpenClaw runtime.

*   `WORKSPACE_DIR`: The absolute path to the directory where OpenClaw will operate.
*   `PORT`: The port on which the OpenClaw dashboard will be accessible.
*   `OLLAMA_URL`: The URL of the local Ollama instance (usually `http://host.docker.internal:11434`).
*   `DEFAULT_AGENT`: The filename (without `.json`) of the agent profile to load by default.
*   `DEFAULT_MODEL`: The name of the Ollama model to use by default.
*   `TELEGRAM_BOT_TOKEN`: (Optional) The token for the Telegram bot.
*   `TELEGRAM_CHAT_ID`: (Optional) The chat ID for Telegram notifications.
